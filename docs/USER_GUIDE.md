# 用户使用指南

## 📖 界面说明

### 运营指挥中心

运营指挥中心是医院管理层的核心可视化平台，提供全局运营视图。

#### 主要功能区域

1. **顶部导航栏**
   - 系统标题
   - 当前日期时间
   - 快捷导航按钮

2. **左侧面板**
   - **实时运营数据**: 显示门诊、急诊、住院、手术、收入等核心指标
   - **科室工作负荷分布**: 柱状图展示各科室工作量对比
   - **医疗设备状态**: 列表展示设备运行状态和使用率

3. **中间面板**
   - **患者流量趋势**: 折线图展示24小时患者流量变化
   - **床位使用情况**: 环形图展示床位占用情况
   - **收入统计分析**: 柱状图展示近7天收入对比

4. **右侧面板**
   - **实时消息预警**: 滚动显示系统预警和重要消息
   - **资源使用监控**: 进度条展示各类资源使用率
   - **科室绩效排名**: 排行榜展示科室绩效

#### 数据指标说明

| 指标名称 | 说明 | 刷新频率 |
|---------|------|---------|
| 门诊人次 | 当日门诊就诊人次 | 3秒 |
| 急诊人次 | 当日急诊就诊人次 | 3秒 |
| 住院人数 | 当前在院患者数 | 3秒 |
| 手术台次 | 当日完成手术台次 | 3秒 |
| 今日收入 | 当日医疗收入统计 | 3秒 |
| 床位使用率 | 床位占用百分比 | 5秒 |

### 急诊科实时看板

急诊科看板专为急诊科设计，提供急诊业务实时监控。

#### 主要功能

1. **顶部统计卡片**
   - 今日急诊人次
   - 等候患者数量
   - 危重症患者数量
   - 床位占用率

2. **急诊患者排队情况**
   - 显示当前排队患者
   - 标注患者优先级（危重/紧急/普通）
   - 显示等待时间

3. **急救车辆追踪**
   - 车辆编号和状态
   - 实时位置信息
   - 车辆可用性

4. **医护人员在岗状态**
   - 医护人员列表
   - 在岗/休息状态
   - 人员岗位信息

5. **危重症患者监控**
   - 患者基本信息
   - 生命体征数据（心率、血压、体温）
   - 异常指标提醒

#### 优先级标识

- 🔴 **危重**: 需要立即处理的紧急情况
- 🟡 **紧急**: 需要优先处理的患者
- 🔵 **普通**: 按顺序等候的患者

### 门诊患者服务大屏

门诊大屏面向患者，提供就医引导和信息服务。

#### 主要功能

1. **叫号信息**
   - 大字号显示当前叫号
   - 提示就诊诊室
   - 自动更新

2. **各科室等候情况**
   - 显示科室名称
   - 当前出诊医生
   - 等候人数
   - 平均等待时间

3. **医生出诊信息**
   - 医生姓名和照片
   - 所属科室
   - 职称信息
   - 出诊状态

4. **就医流程指引**
   - 步骤化展示就医流程
   - 清晰的流程说明
   - 帮助患者了解就医步骤

5. **健康宣教**
   - 健康知识普及
   - 疾病预防提醒
   - 用药安全提示

## 🎮 操作说明

### 页面导航

#### 方式一：使用导航按钮

在运营指挥中心页面，点击右上角的导航按钮：
- 点击"急诊看板"进入急诊科界面
- 点击"门诊大屏"进入门诊服务界面

#### 方式二：使用返回按钮

在急诊或门诊页面，点击左上角"← 返回"按钮回到运营指挥中心。

#### 方式三：直接访问URL

- 运营指挥中心: `http://localhost:5173/`
- 急诊看板: `http://localhost:5173/emergency`
- 门诊大屏: `http://localhost:5173/outpatient`

### 数据刷新

系统采用自动刷新机制，无需手动操作：
- **运营数据**: 每3秒自动更新
- **床位数据**: 每5秒自动更新
- **消息预警**: 每10秒推送新消息

### 全屏显示

建议在大屏展示时使用全屏模式：

**Windows**: 按 `F11` 进入/退出全屏
**Mac**: 按 `Cmd + Ctrl + F` 进入/退出全屏

## 🔍 数据解读

### 趋势分析

#### 患者流量趋势图

- **上升趋势**: 表示就诊人数增加，需要关注资源配置
- **下降趋势**: 表示就诊人数减少，可能是非就诊高峰期
- **波动幅度**: 反映就诊人数的稳定性

#### 科室工作负荷

- **高负荷科室**: 柱状图较高，需要重点关注人员和资源配置
- **低负荷科室**: 柱状图较低，可考虑资源调配
- **负荷对比**: 帮助平衡各科室工作量

### 预警信息

#### 预警级别

- 🔴 **危险**: 需要立即处理的紧急情况
- 🟡 **警告**: 需要关注的异常情况
- ℹ️ **信息**: 一般性通知信息
- ✅ **成功**: 正常完成的操作提示

#### 常见预警

1. **患者激增**: 提醒增派人手
2. **库存不足**: 提醒及时补货
3. **设备故障**: 通知维修处理
4. **床位紧张**: 提示资源调配

### 绩效指标

#### 科室绩效排名

排名基于多维度评估：
- 工作量完成情况
- 患者满意度
- 医疗质量指标
- 运营效率

🥇🥈🥉 前三名科室会有特殊标识

## 💡 最佳实践

### 大屏部署建议

1. **显示器选择**
   - 推荐4K分辨率显示器
   - 最低1080P分辨率
   - 支持触摸屏交互

2. **放置位置**
   - 运营指挥中心：院长办公室、会议室
   - 急诊看板：急诊科护士站
   - 门诊大屏：门诊大厅显眼位置

3. **环境要求**
   - 光线适中，避免强光直射
   - 视角范围内无遮挡
   - 网络连接稳定

### 使用建议

1. **定期查看**
   - 院领导每日查看运营指挥中心
   - 科室主任关注相关科室数据
   - 值班人员实时监控预警信息

2. **数据分析**
   - 关注趋势变化，发现规律
   - 对比历史数据，评估改进
   - 结合实际情况，做出决策

3. **响应处理**
   - 及时响应预警信息
   - 根据数据调整资源配置
   - 记录异常情况和处理结果

## ❓ 常见问题

### Q1: 数据不更新怎么办？

**A**: 检查以下几点：
1. 确认后端服务是否正常运行
2. 检查浏览器控制台是否有错误
3. 刷新页面（F5）重新加载
4. 检查网络连接是否正常

### Q2: WebSocket连接失败？

**A**: 
1. 确认后端服务已启动
2. 检查防火墙设置
3. 验证端口3000是否可访问
4. 查看浏览器控制台错误信息

### Q3: 图表显示异常？

**A**: 
1. 清除浏览器缓存
2. 使用Chrome或Edge浏览器
3. 确保浏览器版本较新
4. 检查是否有JavaScript错误

### Q4: 如何调整刷新频率？

**A**: 
修改后端 `utils/realtime.js` 文件中的时间间隔参数：
```javascript
setInterval(() => { ... }, 3000)  // 3秒改为需要的毫秒数
```

### Q5: 可以自定义显示内容吗？

**A**: 
可以！修改对应的Vue组件文件即可：
- 修改统计指标：编辑 `OperationStats.vue`
- 调整图表样式：修改对应组件的ECharts配置
- 添加新功能：创建新组件并在Dashboard中引入
